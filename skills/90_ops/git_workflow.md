# Skill: Git Workflow (GSMU)

## Objetivo
Trabajo consistente en Git/GitHub.

## Reglas
- No commitear `.env`, `node_modules`, `dist`
- Commits pequeños y descriptivos
- Taggear releases (`vX.Y.Z`)

## Branching sugerido
- `main`: estable
- `dev`: integración
- `feature/*`: desarrollo

## Checklist PR (interno)
- [ ] corre
- [ ] no rompe features existentes
- [ ] README actualizado si cambia setup
